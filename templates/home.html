Key features

Rapid product definition and deployment

Define your products in a simple text file format and they automatically display on web forms.

Profile cluster analysis based instant quotes

Identify common profiles, ask potential customers to select their closest match and provide an instant indicative quote, then adjust details for a final quote. We can show you how to run cluster analysis on your existing data to find those profiles.

Rapid web form and widget customisation

Use the standard forms, include or exclude fields, specify UI widgets used, provide default values.

Automatic back-office administration interface

Auto-generated set of pages for administrators to see and adjust all data.

Easily configured rating engine integration

Outputs an easily configurable XML message of any new risk for consumption by your existing rating engines.

Geocoding and Google maps integration

All addresses entered are automatically geocoded and the geocodes stored. View all of your insured risks on a google map. Roll over the dropped pins for a summary of the risk.

User registration and authentication

Allow users to sign up and save quotes, (optionally) send an email with a unique activation link to validate email addresses of registering users.

Simple white labelling

Run multiple (cosmetically different) sites against the same user and product databases, multi-language facilities to change to another language or just slightly different wording

Examples and Design Patters

Last but certainly not least (if you are a developer) we provide some examples and design patterns to show you how to get started launching insurance services.

How it works

Harnesses the best of the web

Insurance in A Box harnesses the Django web development framework and incorporates Blueprint CSS and jQuery.

Adds some useful bits and pieces

We develop and incorporated extensions for things like XML serialisation, Google maps integration and social network integration. Its not rocket science but who wants to have to maintain rockets? You just need to get new and better insurance products out to the web faster than your competition.

Keep the data architecture simple

For each new product defined Insurance in A Box (via Django) creates a new set of relational tables, linked into the existing user tables. No SQL coding required and no trying to shoehorn new products into old schemas (it never works anyway).
